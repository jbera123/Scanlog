二、CSV 文件格式
----------------

每个文件包含两列：

    epc_hex,label
    AA000001,ZB
    AA000002,ZB
    ...

- epc_hex：要写入 RFID 标签的 EPC 编码（每个标签唯一，不可重复）。
- label ：标签表面需要印刷的文字（同一 SKU 全部相同）。

第一行为表头，请勿烧录或印刷。


三、EPC 编码技术规范
--------------------

1. 协议：EPC Gen2 (ISO 18000-6C)，UHF 860–960 MHz。
2. 存储区 (Memory Bank)：EPC 区（Bank 01）。
3. 起始地址：字地址 02（即跳过 CRC 和 PC 字，从 EPC 本体开始写入）。
4. EPC 长度：96 位（6 个字 / 24 个十六进制字符）。
   CSV 中每个 epc_hex 为 8 个十六进制字符，请在其右侧补 16 个 0
   凑足 96 位。例如：
       CSV 值：    AA000001
       实际烧录：  AA000001 0000000000000000
5. PC 字：设置为 0x3000（表示 EPC 长度 96 位，无 UMI，无 XPC）。
6. 字符集：仅使用 0–9 和 A–F（大写），不使用小写。
7. 锁定 (Lock)：不锁定。出厂后客户端可覆写。
8. 访问密码 / 销毁密码：全部保持默认 0x00000000。


四、标签面印刷
--------------

- 每个标签印刷的文字即 CSV 中 label 列的值（如 ZB、KB、L-FS 等）。
- 同一 SKU 的所有标签印刷文字完全相同，不需要把 EPC 编号印到标签面上。
- 字体、字号、颜色、条码等请沿用此前确认的版型。
- 注意：印刷文字与 CSV 文件名中的 SKU 不一定相同，请以 label 列为准。


五、唯一性与质检要求
--------------------

1. 每个 epc_hex 在全批 1200 个标签中必须唯一，严禁重复。
2. 每个 SKU 的烧录数量必须与本说明"文件清单"中的数量一致。
3. 烧录完成后请抽样 5% 读取验证，确认：
   a. EPC 与 CSV 中对应行一致；
   b. PC 字为 0x3000；
   c. 标签可被普通 UHF 读写器识别。
4. 如发现坏标签，请单独标记并补烧一个新的 epc_hex
   （使用 CSV 末尾未使用的序号，例如 AA000301、AA000302 …），
   不要复用已报废的编号。

--------

- 有任何疑问请在烧录前联系我方确认，勿自行修改编码规则。
